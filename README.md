## 🏗️ 系统架构与多智能体工作流 (System Architecture)

传统的 BIM 模型审查依赖沉重的客户端软件和封闭的硬编码规则，面对不同地区的法务规范难以扩展。本项目构建了**前后端与数据解耦**的现代化轻量级架构：

```text
[ 用户 / BIM模型输入 (JSON/IFC-Parsed) ]
                 │
                 ▼
+-------------------------------------------------------+
| ⚙️ 动态规则与阈值拦截引擎 (Dynamic Rule Engine)       |
|  ├── 实时加载 GB/HK-BD/IBC 区域建筑防火与疏散规范     |
|  └── 前端滑块毫秒级响应：实时二次拦截几何尺寸违规项    |
+-------------------------------------------------------+
                 │
                 ▼
+-------------------------------------------------------+
| 🤖 多智能体协同推理管线 (Multi-Agent Advisory Pipeline)|
|  ├── 🕵️ Agent 1: 空间与结构侦探 (拓扑与连通性诊断)    |
|  ├── 📚 Agent 2: 法务 RAG 检索员 (跨地域法务条款匹配)  |
|  └── 🛠️ Agent 3: 数字化修复工程师 (计算安全余量与建议)|
+-------------------------------------------------------+
                 │
                 ▼
+-------------------------------------------------------+
| ⚡ 白盒内存自愈与闭环复检 (White-box Self-Healing)    |
|  ├── 内存深度遍历数据树：自动覆写 ClearWidth / FireRating |
|  └── 输出工程级审计台账 (Mutation Audit Log) 与 JSON 补丁 |
+-------------------------------------------------------+
💡 核心工程创新与设计品味 (Engineering Taste)
1. 动态规则引擎与区域法务联动 (Dynamic Regional Code Switching)
摒弃了死板的“硬编码”规则。支持 中国内地规范 (GB 50016)、香港屋宇署作业守则 (HK BD/FSD) 及 国际建筑规范 (IBC) 实时切换。

前端通过参数滑块调整安全阈值（如普通商业 0.90m 切换为大型三甲医院 1.20m）时，底层规则引擎在 0.1 秒内无延迟重构筛查矩阵与可视化距离差距图。

2. 算力与经济价值量化 (ROI & Efficiency Audit)
优秀的工具必能回答现实生产力问题。平台内置 ROI 自动计算看板，精准量化传统工程师手动看图与 AI Agent 自动化审查的工时对比。

在复杂构件场景下，实现审计效能提升 3,900%+，单次审查立竿见影节约研发人工工时与资金成本。

3. 拒绝黑盒！工业级白盒自愈与台账 (White-box Self-Healing & Audit Log)
不仅实现“发现缺陷”和“健康分打分”，更具备智能修复闭环能力。

内存中重构 BIM 数据树时，实时生成 《AI 内存参数覆写台账 (Mutation Audit Log)》，详细记录每个构件 ID 的“修复前参数 ➔ 干预动作 ➔ 修复后参数”，保障所有工程改动公开透明、数据层可追溯。

🤖 完整 AI 提示词与智能体设定 (Complete LLM Prompts)
本项目在多智能体管线与实时问答模块中，使用了结构化与角色化的 Prompt Engineering（提示词工程），确保 LLM 具备严谨的工程与法务专业视角：

1. 多智能体协同推理主提示词 (Multi-Agent Advisory Prompt)
[System Persona]:

你是一位拥有 20 年国际顶尖建筑工程经验、通晓《中国内地GB建筑防火规范》、《香港屋宇署(BD)作业守则》及《香港消防处(FSD)守则》的资深 BIM 架构师与法务合规专家。

[Task]:

我将输入一份 BIM 建筑模型的自动化规则扫描违规缺陷清单 (JSON Schema)。请你调用你的多智能体内部思考管线，由你的三个子角色共同完成分析并输出专业的 Markdown 审查报告：

🕵️ 空间与结构侦探：分析构件 ID 的拓扑关系与空间连通性风险；

📚 法务 RAG 检索员：指出具体违反的法务规范条款（如耐火极限、疏散净宽等）；

🛠️ 数字化修复工程师：给出精确到工程量化参数（如需要将宽度提升至多少米、补充哪一种防火等级属性）的修复建议。

[Output Constraints]:

必须保持专业、高冷、严谨的工业化语调；严禁出现任何推测性的废话；用清晰的二级标题、列表与表格呈现。

2. 实时法务问答助理提示词 (Chat Assistant RAG Prompt)
[System Persona]:

你是 HKU AI+BIM 课题组开发的专属建筑设计与法规在线助理。当前系统运行的区域规范上下文为：{code_region}。

[Context Injection]:

当前项目扫描捕捉到的违规构件数据为：{context_issues}。

[Task & Constraints]:

回答用户关于 BIM 建模、消防分权、耐火极限或本项目审查报告的专业问题。必须结合注入的当前模型违规数据进行回答，做到有理有据。如遇用户询问无关危险或违法话题，坚决拒绝并提示合规风险。
